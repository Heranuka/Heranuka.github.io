Comment Tree — сервис иерархических комментариев

Разработал сервис для хранения и выборки вложенных комментариев с фокусом на производительность запросов.

  • Выбрал схему хранения древовидных комментариев, позволяющую эффективно работать с вложенными ветками.
  • Оптимизировал выборки для сценариев: загрузка ветки, получение последних комментариев, вывод длинных тредов с пагинацией.
  • Добавил сортировку и поиск по тексту, чтобы было проще работать с большими лентами обсуждений.
  • Продумал API так, чтобы им было удобно пользоваться как фронтенду, так и внешним сервисам.

Стек: Go, PostgreSQL, Docker.

GitHub: comment-tree

Warehouse Control — система учёта складских операций

Реализовал компактную систему управления запасами с прозрачным аудитом действий пользователей.

  • Настроил аутентификацию по JWT и разграничение прав через роли (администратор, менеджер, сотрудник склада).
  • Сделал детальный журнал действий: кто, когда и какие изменения внёс в карточки товаров и остатки.
  • Спроектировал модель данных так, чтобы поддерживать историю изменений по складам и пользователям.
  • Реализовал базовые защитные механизмы: ограничение по числу запросов и валидацию входных данных на уровне API.

Стек: Go, PostgreSQL, JWT, Docker.

GitHub: warehouse-control

Image Processor — сервис фоновой обработки изображений

Сконструировал сервис, который асинхронно обрабатывает изображения и подготавливает их для дальнейшего использования.

  • Использовал Apache Kafka для очередей заданий на обработку изображений и разгрузки основного API.
  • Реализовал операции изменения размера и генерации превью с сохранением артефактов в персистентное хранилище.
  • Отделил слой обработки от слоя приёма запросов, чтобы можно было масштабировать воркеры независимо.
  • Добавил валидацию входных данных (формат, размер файлов) и обработку типичных ошибок чтения/записи.

Стек: Go, PostgreSQL, Kafka, Docker.

GitHub: image-processor

Order Service — микросервис управления заказами

Спроектировал и реализовал сервис обработки заказов с акцентом на разделение доменной логики и инфраструктуры.

  • Реализовал обмен доменными событиями по Apache Kafka для связи сервиса заказов с платежами и складом.
  • Разделил код на уровни domain, application и infrastructure, что позволило писать модульные тесты без поднятия внешних сервисов.
  • Описал контракт взаимодействия через чёткие DTO и явные topic’и, что упростило интеграцию с другими командами.
  • Настроил конфигурацию и подключение к Kafka и БД через переменные окружения и Docker‑компоуз.

Стек: Go, PostgreSQL, Kafka, Redis, Docker.

GitHub: order-service

Delayed Notifier — сервис планирования уведомлений

Разработал сервис для отправки уведомлений по расписанию с контролем статуса доставки и повторными попытками.

  • Настроил использование RabbitMQ с плагином delayed message exchange для запуска уведомлений точно в заданный момент.
  • Спроектировал механизм повторных отправок с увеличивающимся интервалом через DLX и отдельные retry‑очереди.
  • Реализовал несколько каналов доставки (Email, Telegram) и управление жизненным циклом уведомлений: создание, перенос, отмена.
  • Организовал обработку задач воркерами с учётом лимитов по количеству одновременных запросов к внешним сервисам.
  • Добавил базовые метрики и логирование для отслеживания статуса доставки и проблемных сообщений.

Стек: Go, PostgreSQL, Redis, RabbitMQ, Docker.

GitHub: delayed-notifier
