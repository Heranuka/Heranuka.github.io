Учебные и pet‑проекты

Comment Tree — сервис иерархических комментариев

Сервис для хранения и выборки вложенных комментариев с фокусом на производительность запросов и удобство работы с длинными тредами.

  • Спроектировал и реализовал схему хранения древовидных данных на основе Materialized Path, что позволило получать ветку комментариев любой вложенности за один SQL‑запрос вместо серии рекурсивных выборок.
  • Настроил индексы и оптимизировал основные запросы для сценариев: загрузка ветки, получение последних комментариев и пагинация длинных обсуждений, снизив нагрузку на БД при массовом чтении.
  • Реализовал API с пагинацией и сортировкой, сократив объём передаваемых данных и упростив работу фронтенда с большими лентами обсуждений.
  • Добавил поиск по тексту комментариев и фильтрацию по параметрам, что упростило навигацию по длинным дискуссиям.

Стек: Go, PostgreSQL, Docker.

GitHub: comment-tree

Warehouse Control — система учёта складских операций

Компактная система управления запасами с прозрачным аудитом действий пользователей и ролевой моделью доступа.

  • Разработал механизм аудита, логирующий все изменения остатков и карточек товаров, что обеспечило прозрачность операций на складе и возможность восстанавливать историю действий по каждому пользователю.
  • Реализовал ролевую модель доступа (администратор, менеджер, сотрудник склада) с аутентификацией по JWT для гранулярного контроля над CRUD‑операциями.
  • Спроектировал модель данных с поддержкой истории изменений по складам и пользователям, включая операции прихода, расхода и корректировок.
  • Добавил защитные механизмы на уровне API: валидацию входных данных и ограничение по числу запросов, снизив риск некорректных операций и базовых DoS‑сценариев.

Стек: Go, PostgreSQL, JWT, Docker.

GitHub: warehouse-control

Image Processor — сервис фоновой обработки изображений

Сервис асинхронной обработки изображений с отделением HTTP‑API от ресурсоёмких операций и независимым масштабированием воркеров.

  • Построил асинхронный пайплайн обработки изображений на базе Apache Kafka, отделив приём запросов от тяжёлых операций и позволив независимо масштабировать воркеры.
  • Реализовал операции изменения размера и генерации превью с сохранением результатов в персистентное хранилище для дальнейшего использования другими сервисами.
  • Вынес слой обработки в отдельный компонент, что упростило горизонтальное масштабирование и запуск отдельных пулов воркеров под разные типы задач.
  • Реализовал graceful shutdown для воркеров, обеспечив корректное завершение текущих задач при остановке сервиса и обработку типичных ошибок чтения/записи файлов.

Стек: Go, PostgreSQL, Kafka, Docker.

GitHub: image-processor

Order Service — микросервис управления заказами

Сервис обработки заказов с акцентом на разделение доменной логики и инфраструктуры и обмен доменными событиями.

  • Применил принципы Clean Architecture и разделение на слои domain, application и infrastructure, что позволило покрыть бизнес‑логику модульными тестами без поднятия БД и брокера сообщений.
  • Спроектировал обмен доменными событиями по Kafka для интеграции сервиса заказов с платёжным сервисом и складом, используя чётко определённые DTO и топики.
  • Реализовал идемпотентные консьюмеры Kafka для обработки доменных событий и защиты от повторной доставки сообщений, обеспечив корректное состояние заказов.
  • Настроил конфигурацию подключения к БД и Kafka через переменные окружения и Docker‑compose, упростив локальный запуск и деплой.

Стек: Go, PostgreSQL, Kafka, Redis, Docker.

GitHub: order-service

Delayed Notifier — сервис планирования уведомлений

Сервис отправки уведомлений по расписанию с контролем статуса доставки, повторными попытками и несколькими каналами доставки.

  • Настроил использование RabbitMQ с плагином delayed message exchange для точного запуска уведомлений в заданный момент без опроса БД и таймеров в приложении.
  • Спроектировал механизм повторных отправок с увеличивающимся интервалом через DLX и отдельные retry‑очереди, повысив устойчивость к временным сбоям внешних сервисов.
  • Реализовал несколько каналов доставки (Email, Telegram) и управление жизненным циклом уведомлений: создание, перенос, отмена и отслеживание статуса.
  • Организовал обработку задач воркерами с учётом лимитов по количеству одновременных запросов к внешним API, снизив риск превышения квот и блокировок.
  • Добавил базовые метрики и логирование для мониторинга статусов доставки и выявления проблемных сообщений.

Стек: Go, PostgreSQL, Redis, RabbitMQ, Docker.

GitHub: delayed-notifier
